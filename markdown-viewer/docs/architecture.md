# Architecture

## Repository Layout

- `markdown-viewer/` -- the Vue app (all source, config, build artifacts)
  - `src/` -- Vue components, composables, lib utilities
  - `lib/` -- shared code used by both Vite plugins and prebuild (story-scanner, R2 client)
  - `vite-plugins/` -- custom dev server API plugins
  - `api/` -- Vercel serverless functions (prod only)
  - `scripts/` -- build scripts (prebuild)
  - `data/` -- gitignored runtime data (images.json)
  - `public/api/` -- gitignored, generated by prebuild
- `example/` -- sample `.md` story files
- Story `.md` files are placed in subdirectories of the repo root

## Dual API Mode

The app has two API modes controlled by `VITE_STATIC_API`:

### Dev mode (`VITE_STATIC_API` unset)

Vite dev-server plugins serve API routes directly from the filesystem and R2:

| Plugin | File | Routes |
|--------|------|--------|
| markdown-api | `vite-plugins/markdown-api.ts` | `/api/stories` (`?fresh=1` for cache bypass), `/api/content` |
| image-api | `vite-plugins/image-api.ts` | `/api/images`, `/api/images/file`, `/api/images/upload`, `/api/images/delete`, `/api/images/delete-character`, `/api/images/delete-all` |

The markdown-api plugin adds the parent directory (repo root) to the Vite file watcher and invalidates its cached story tree on `.md` file or directory changes. The frontend triggers a cache-bypassing re-scan (`?fresh=1`) when a sidebar directory is expanded.

### Static/production mode (`VITE_STATIC_API=true`)

The `prebuild` script (`scripts/prebuild.ts`) generates static files:
- `public/api/stories.json` -- story tree
- `public/api/content/**/*.md` -- copied markdown files
- `public/api/images.json` -- image metadata

Only `/api/images/file` remains dynamic, handled by a Vercel serverless function (`api/images/file.ts`).

The client switches endpoints in `src/lib/api.ts` based on `import.meta.env.VITE_STATIC_API`.

## Markdown Rendering Pipeline

`src/lib/markdown.ts` uses `marked` to lexer raw Markdown into tokens, then transforms them into a custom typed AST defined in `src/lib/types.ts`. Vue components in `src/components/content/` render this AST.

Key behaviors:
- Single-image paragraphs are promoted to `figure` nodes
- URLs are sanitized to allow only `http:`, `https:`, relative paths, and hash links
- Adjacent text nodes are merged for cleaner output
- GFM and line breaks are enabled

## Image Management

Character images are stored in Cloudflare R2, organized as `character/outfit/filename`. Metadata is tracked in `data/images.json` (gitignored). The image API plugin (dev) and serverless function (prod) both use the shared R2 client in `lib/r2.ts`.

Required env vars: `R2_ACCOUNT_ID`, `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_BUCKET_NAME`.

Validation rules:
- Max file size: 10MB
- Max files per upload: 50
- Allowed extensions: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`, `.avif`
- Path segments are sanitized to alphanumeric, CJK characters, `.`, `_`, `-`

## Story Scanner

`lib/story-scanner.ts` recursively scans the **repo root** (parent of `markdown-viewer/`) for `.md` files. Directories named `markdown-viewer` and `node_modules` are excluded, as are hidden directories (starting with `.`). Root-level `.md` files are skipped (only subdirectory contents are collected). Results are sorted with natural ordering.

## Authentication

`middleware.ts` (Vercel middleware) enforces Basic Auth on all routes using `BASIC_AUTH_USER` and `BASIC_AUTH_PASSWORD` env vars with constant-time comparison.

## UI Stack

- **shadcn-vue** (new-york style) via reka-ui primitives -- `src/components/ui/`
- **Tailwind CSS v4** via `@tailwindcss/vite` plugin
- **Lucide** icons via `lucide-vue-next`
- **Composables** in `src/composables/` follow Vue 3 Composition API patterns

## Key Conventions

- Path alias: `@/` maps to `src/`
- TypeScript project references: `tsconfig.app.json` (browser), `tsconfig.node.json` (Node/Vite)
- Linting: eslint with vue, typescript, sonarjs, unicorn, security, import-x, regexp, better-tailwindcss plugins; Prettier rules via eslint-config-prettier
- `data/` is gitignored (local-only runtime data)
- `public/api/` is gitignored (generated by prebuild)

## Environment Variables

See `.env.example`:

| Variable | Purpose |
|----------|---------|
| `R2_ACCOUNT_ID` | Cloudflare R2 account ID |
| `R2_ACCESS_KEY_ID` | R2 API token access key |
| `R2_SECRET_ACCESS_KEY` | R2 API token secret key |
| `R2_BUCKET_NAME` | R2 bucket name (default: `character`) |
| `BASIC_AUTH_USER` | Vercel middleware auth user |
| `BASIC_AUTH_PASSWORD` | Vercel middleware auth password |
| `VITE_STATIC_API` | Set `true` for static API mode (production build) |
